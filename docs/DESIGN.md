## 1. 项目概述

本系统面向移动端用户，通过上传人脸照片完成签到。服务端利用视觉神经网络进行人脸检测与特征编码，在底库中检索匹配身份并记录签到。系统分为三个核心模块：

| 模块 | 是否需要登录 | 说明 |
|------|-------------|------|
| 照片库管理 | 是 | 注册/录入人脸、维护底库 |
| 签到 | 否 | 上传照片 → 检索身份 → 记录签到 |
| 签到记录查看 | 是 | 查询历史签到记录 |

### 2.2 核心数据流

#### 签到流程（无需登录）

```
用户拍照 → 上传图片 → 服务端接收
    → RetinaFace 检测人脸 → ArcFace 提取 512 维特征向量
    → FAISS 在底库中 1:N 检索 → 余弦相似度 > 阈值则匹配成功
    → 写入签到记录 → 返回「签到成功 + 姓名」
```

#### 照片库录入流程（需登录）

```
管理员登录 → 上传用户照片 + 姓名/工号
    → 检测 + 编码 → 存入 PostgreSQL (元数据) + FAISS 索引 (向量)
```

### 2.3 高并发设计

| 策略 | 方案 |
|------|------|
| API 层 | FastAPI 异步 I/O；多 Worker (Uvicorn + Gunicorn) |
| 人脸推理 | 独立推理进程池 / 可选 GPU 加速；请求队列削峰 |
| 向量检索 | FAISS IndexFlatIP / IVF 索引，内存常驻，毫秒级检索 |
| 缓存 | Redis 缓存热点用户特征、签到限流（同用户 N 秒内不可重复签到） |
| 数据库 | PostgreSQL 连接池 (asyncpg)；读写分离（扩展阶段） |
| 负载均衡 | Nginx 反向代理 + 多实例水平扩展 |

## 3. 模型选型

### 3.1 人脸检测

| 模型 | 优点 | 缺点 | 选用 |
|------|------|------|------|
| **MTCNN** | 经典、轻量、易部署 | 精度与速度已落后于新模型 | 备选 |
| **RetinaFace** | 精度高、多尺度检测、InsightFace 内置 | 模型略大 | **主选** |
| SCRFD | 速度极快、移动端友好 | 需额外集成 | 备选 |

**选定：RetinaFace**（通过 InsightFace 库统一调用，与特征提取模型配套）

### 3.2 人脸特征编码

| 模型 | 特征维度 | 优点 | 缺点 | 选用 |
|------|---------|------0------|------|
| **FaceNet** | 128/512 | 经典、文档丰富 | 精度略低于新模型 | 备选 |
| **ArcFace** | 512 | SOTA 精度、InsightFace 原生支持 | 需 GPU 更佳 | **主选** |
| dlib (ResNet) | 128 | 部署简单 (face_recognition) | 精度一般 | 教学备选 |

**选定：ArcFace (r100, MS1MV3 预训练)**，512 维 L2 归一化向量，余弦相似度匹配。

### 3.3 匹配策略

- **相似度度量**：余弦相似度（向量已 L2 归一化，等价于内积）
- **默认阈值**：0.45 ~ 0.55（可通过验证集标定，框架默认 0.5）
- **检索引擎**：FAISS `IndexFlatIP`（底库 < 10 万）；规模: `IndexIVFFlat`（更大规模）

### 3.4 技术栈汇总

| 层次 | 技术 |
|------|------|
| 移动端 | HTML5 + Vue 3（H5，浏览器拍照上传） |
| 后端 | Python 3.11 + FastAPI |
| 人脸识别 | InsightFace (RetinaFace + ArcFace) |
| 向量检索 | FAISS |
| 数据库 | PostgreSQL 15 |
| 缓存 | Redis 7 |
| 部署 | Docker Compose / 云服务器公网 IP |

## 4. 数据库设计

### 4.1 ER 关系

```
User (管理员)          FaceProfile (人脸底库)         CheckInRecord (签到记录)
┌──────────────┐      ┌──────────────────┐         ┌─────────────────────┐
│ id           │      │ id               │         │ id                  │
│ username     │      │ name             │         │ face_profile_id (FK)│

│ created_at   │      │ photo_path       │         │ similarity_score    │
```

| 字段 | 类型 | 说明 |

**check_in_records** — 签到记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| face_profile_id | FK | 匹配到的人脸 |

## 5. API 设计

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/auth/login` | 否 | 管理员登录，返回 JWT |
| POST | `/api/faces` | 是 | 录入人脸（姓名 + 工号 + 照片） |
| GET | `/api/faces` | 是 | 列出底库 |
| DELETE | `/api/faces/{id}` | 是 | 删除/禁用人脸 |
| POST | `/api/check-in` | 否 | 上传照片签到 |
| GET | `/api/check-in/records` | 是 | 查询签到记录（支持日期筛选） |
| GET | `/api/health` | 否 | 健康检查 |

### 签到接口响应示例

```json
// 成功
{ "success": true, "name": "张三", "employee_no": "2021001", "message": "签到成功" }

// 失败
{ "success": false, "message": "未识别到匹配的人脸，请重试" }
```

## 7. 部署与运行

1. 云服务器开放公网 IP，防火墙放行 80/443 端口
2. `docker compose up -d` 启动 PostgreSQL、Redis、API 服务
3. 首次启动自动下载 InsightFace 模型权重
4. 移动端浏览器访问 `http://<公网IP>/` 进行签到
